# Рабочая очередь

Последнее обновление: 24 июля 2026 года.

Этот файл показывает порядок работы, а не заменяет подробный roadmap. Приоритет
новой крупной функции подтверждает владелец проекта.

## Формат номера

- `STUDIO-XXX` — пользовательский интерфейс Studio;
- `ORDER-XXX` — заявка, заказ и производственный пакет;
- `SITE-XXX` — публичная страница;
- `SEC-XXX` — безопасность и проверка файлов;
- `QA-XXX` — тестирование, доступность и производительность;
- `TECH-XXX` — архитектура и инфраструктура;
- `DOCS-XXX` — документация и взаимодействие команды.

Номер задачи не переиспользуется после завершения или отмены.

## Как двигать задачу

1. При начале перенести её в «Сейчас» и указать рабочий статус.
2. Одновременно выполнять только одну крупную задачу на участника.
3. После локальной готовности указать пройденные и непройденные проверки.
4. После merge перенести задачу в «Готово», добавить дату и номер PR.
5. Отменённую задачу оставить с причиной и статусом `отменено`.
6. Новую задачу добавлять с ближайшим свободным номером своей категории.
