# Текущее состояние проекта

Последнее обновление: 24 июля 2026 года.

Этот файл отвечает на вопрос «что происходит с проектом сейчас». Устойчивая
карта архитектуры находится в [PROJECT_CONTEXT.md](PROJECT_CONTEXT.md), а
долгосрочный порядок развития — в [ROADMAP.md](ROADMAP.md).

## Production

- Основная ветка: `main`.
- Последний опубликованный пользовательский результат: исправление геометрии
  мобильного предпросмотра и равных интервалов повторов ленты.
- Production: `https://печатаетмаксим.рф/`.
- Studio: `https://печатаетмаксим.рф/studio/`.
- Hosting: REG.RU через `.github/workflows/deploy-reg-ru.yml`.
- Точный опубликованный commit определять по `main` и последнему успешному
  запуску workflow `Deploy to REG.RU`, а не по сохранённому здесь SHA.

## Текущий этап развития

- В проект внедрена единая система передачи контекста человеку и ChatGPT:
  - `docs/PROJECT_CONTEXT.md`;
  - этот файл;
  - журнал решений и рабочая очередь;
  - шаблон задачи и GitHub Issue;
  - обязательные правила актуализации в `AGENTS.md`;
  - шаблон Pull Request;
  - автоматическое предупреждение о забытой документации.
- Следующий крупный продуктовый этап — реальная защищённая доставка заявки.

## Состояние Studio

- Рабочий production-runtime:
  `geometry.js → layout.js → app.js → mobile-products.js`.
- Шаги:
  - 01 — общий текст и логотип;
  - 02 — индивидуальные настройки ленты и стикера;
  - 03 — состав заказа и локальный файл заявки.
- Поддерживаются SVG, PNG, JPEG и первая страница PDF.
- Есть crop, автоматическая одноцветная трассировка, отдельные стили изделий,
  автоматический/ручной layout и мобильный плавающий предпросмотр.
- Контрольные размеры тестов: `390 × 844` и `1440 × 900`.
- Заявка пока не отправляется на сервер.

## Ближайшие приоритеты

1. Подтвердить продуктовые решения для `ORDER-001`.
2. Добавить строгую проверку и лимиты загружаемых файлов (`SEC-001`).
3. Разделить Playwright-тесты по сценариям (`TECH-001`).
4. Добавить WebKit, accessibility и визуальные regression-проверки.

Подробности и критерии готовности находятся в `docs/ROADMAP.md`.

## Известные технические ограничения

- `studio/assets/js/app.js`, CSS и основной тест Studio слишком велики.
- `studio/components/` не подключён к production.
- Клиентская очистка SVG требует усиления.
- Crop и трассировка работают в основном потоке.
- CI ориентирован на Chromium.
- Нет backend, `request_id` и подтверждённой доставки заявки.

## Что проверить перед новой задачей

1. Выполнить `git fetch origin`.
2. Проверить актуальный `origin/main`.
3. Убедиться, что чужие локальные изменения не затрагиваются.
4. Прочитать `AGENTS.md` и `docs/PROJECT_CONTEXT.md`.
5. Уточнить режим: анализ, локальная реализация или публикация.

## Как обновлять этот файл

Обновлять в той же задаче, когда:

- опубликована новая версия;
- изменился текущий крупный этап;
- появилась или снята известная проблема;
- работа передаётся другому участнику;
- изменилась активная ветка длительной работы;
- изменились ближайшие приоритеты.

Не перечислять здесь каждую мелкую CSS-правку. Для истории завершённых изменений
использовать `docs/CHANGELOG.md` и GitHub.
