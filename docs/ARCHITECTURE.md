# Архитектура Studio

## Назначение

Studio — основной интерактивный интерфейс проекта «Печатает Максим» для подготовки фирменной ленты и стикеров.

## Основной каталог

Рабочие файлы Studio находятся в `studio/`.

## Текущая структура

- `studio/index.html` — основная HTML-разметка интерфейса.
- `studio/assets/css/app.css` — стили и адаптивное представление.
- `studio/assets/js/app.js` — клиентская логика и управление состоянием интерфейса.
- `studio/assets/images/` — изображения и фирменные графические материалы.
- `studio/components/` — заготовки модульных компонентов.
- `studio/data/product.json` — описание продукта и его возможностей.

## Текущая модель приложения

- Studio работает в браузере как клиентское приложение.
- Основной runtime подключается через `studio/assets/js/app.js`.
- Состояние интерфейса, загрузка файлов, обработка изображений и предпросмотр выполняются на стороне клиента.
- SVG используется для динамического отображения логотипа, текста, ленты и стикера.

## Архитектурные ограничения

- Нельзя удалять существующую рабочую функциональность без явного требования.
- Изменения должны быть локальными и затрагивать минимальное количество файлов.
- Mobile First изменения не должны ломать Desktop на ширине `1440 px`.
- Контент предпросмотра должен оставаться внутри физических границ ленты и стикера.

## Документирование решений

Значимые изменения структуры, потока данных или способа рендеринга следует добавлять в этот документ после их реализации и проверки.
